Mecânicas do Jogo

1

Processamento Anual

A cada ano, emoções com intensidade baixa (< 30) são removidas, simulando o "esquecimento" de sentimentos triviais.

2

Ruminação

Emoções dominantes podem gerar novos sentimentos similares, reforçando padrões emocionais.

3

Colapso Emocional

Se qualquer emoção exceder 150 de intensidade, o personagem sofre colapso e o jogo termina.

4

Escolhas Estratégicas

Cada ano apresenta eventos adaptativos para a idade, com 4 opções que moldam o desenvolvimento.

Árvore Rubro-Negra

Definição

Uma árvore binária balanceada onde cada nó é colorido de vermelho ou preto. Garante operações de busca, inserção e remoção em O(log n), mesmo nos piores casos.

Propriedades

  • ● Raiz é sempre PRETA
  • ● NIL são PRETOS
  • ● Pai de vermelho é preto
  • ● Todos os caminhos até NIL têm mesmo número de pretos

Inserção

O(log n)

Insere novo nó e rebalanceia a árvore mantendo as propriedades.

Busca

O(log n)

Encontra emoções com intensidade específica rapidamente.

Remoção

O(log n)

Remove emoções triviais e rebalanceia a árvore.

Rotações

O(1)

Operações locais que mantêm o balanceamento.

Uso no Simulador

1

Armazena emoções com suas intensidades ordenadas

2

Permite encontrar emoção dominante em O(1)

3

Remove emoções triviais eficientemente

4

Mantém estrutura sempre balanceada

Como Instalar e Executar

1

Clone o Repositório

git clone https://github.com/EDAII/ArvoreBalanceada_RedBlack.git
2

Navegue para a Pasta

cd ArvoreBalanceada_RedBlack
3

Compile com GCC

gcc main.c RedBlack.c -o rodar
4

Execute o Programa

./rodar

Requisitos

Linux / Windows com GCC
GCC 9.4 ou superior
Padrão C99
Sem dependências externas

Usuários Windows: Instale o GCC ou use compiladores online como OnlineGDB
